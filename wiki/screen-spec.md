# 画面仕様書

## 対象読者

開発者・QAエンジニア

---

## 1. ログイン画面（index.html）

### 概要

アプリケーションのエントリーポイント。メールアドレスとパスワードで認証を行い、成功時にタスク一覧画面へ遷移する。

### URL

- ローカル：`http://localhost:5173/index.html`
- GitHub Pages：`https://tklm14.github.io/demo-qa-portfolio/`

### 画面要素

| 要素 | data-testid | 種別 | 説明 |
|------|-------------|------|------|
| エラーメッセージ領域 | `error-message` | div | バリデーション・認証失敗時に表示。初期状態は `hidden` |
| メールアドレス入力欄 | `email-input` | input[text] | プレースホルダー：`demo@example.com` |
| パスワード入力欄 | `password-input` | input[password] | プレースホルダー：`パスワード` |
| ログインボタン | `login-button` | button[submit] | フォーム送信をトリガー |

### バリデーション仕様

送信ボタンクリック時に以下の順序でバリデーションを実行する。いずれかに引っかかった時点で処理を止めエラーメッセージを表示する。

| 順序 | 対象 | 条件 | エラーメッセージ |
|------|------|------|----------------|
| 1 | メールアドレス | 空文字（trim後） | `メールアドレスを入力してください` |
| 2 | メールアドレス | 正規表現 `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` に不一致 | `正しい形式で入力してください` |
| 3 | パスワード | 空文字 | `パスワードを入力してください` |
| 4 | 認証照合 | デモアカウントと不一致 | `メールアドレスまたはパスワードが正しくありません` |

### 遷移仕様

| 条件 | 遷移先 |
|------|--------|
| 認証成功 | `tasks.html` |
| バリデーション・認証失敗 | 同画面（エラーメッセージを表示） |

## 2. タスク一覧画面（tasks.html）

### 概要

ログイン後のメイン画面。タスクの追加・完了切り替え・削除・検索を行う。未ログインの場合は `index.html` へリダイレクトする。

### URL

- ローカル：`http://localhost:5173/tasks.html`

### アクセス制御

画面ロード時に `isAuthenticated()` を確認する。`false` の場合は `index.html` へリダイレクトする（ブラウザバック経由でもアクセス不可）。

### 画面要素

| 要素 | data-testid | 種別 | 説明 |
|------|-------------|------|------|
| タスクタイトル入力欄 | `task-title-input` | input[text] | プレースホルダー：`タスクタイトルを入力（50文字以内）` |
| 追加ボタン | `add-task-button` | button | クリックでタスクを追加 |
| バリデーションエラー | `validation-error` | div | 追加失敗時に表示。初期状態は `hidden` |
| 検索入力欄 | `search-input` | input[text] | リアルタイムフィルタリング |
| 空メッセージ | `empty-message` | p | 0件時に表示。初期状態は `hidden` |
| タスクリスト | `task-list` | ul | タスク一覧の描画コンテナ |
| タスクアイテム | `task-item` | li | 各タスクの行。`data-task-id` にタスクIDを保持 |
| チェックボックス | `task-checkbox` | input[checkbox] | 完了状態の切り替え |
| タスクタイトル | `task-title` | span | タスク名。完了時は `.completed` クラスが付与される |
| 編集ボタン | `edit-task-button` | button | タスク編集画面へ遷移 |
| 削除ボタン | `delete-task-button` | button | タスクを削除 |
| ログアウトボタン | `logout-button` | button | 認証情報を削除してログイン画面へ遷移 |

### タスク追加バリデーション

追加ボタンクリック時に以下を確認する。

| 条件 | エラーメッセージ |
|------|----------------|
| タイトルが空文字（trim後） | `タイトルを入力してください` |
| タイトルが51文字以上 | `タイトルは50文字以内で入力してください` |

### 検索仕様

- 検索入力欄への入力をトリガーにリアルタイムでフィルタリングする
- タイトルの部分一致で絞り込む
- **大文字・小文字は区別しない**（toLowerCase で比較）
- 検索クリア時は全件表示に戻る
- 0件の場合は `empty-message` を表示する

### 遷移仕様

| 操作 | 遷移先 |
|------|--------|
| 編集ボタンクリック | `task-detail.html?id={taskId}` |
| ログアウトボタンクリック | `index.html` |
| 未ログインでアクセス | `index.html`（リダイレクト） |

## 3. タスク編集画面（task-detail.html）

### 概要

既存タスクの詳細を表示・編集する画面。URLのクエリパラメータ（`?id=`）でタスクを特定する。

### URL

- ローカル：`http://localhost:5173/task-detail.html?id={taskId}`

### アクセス制御

| 条件 | 動作 |
|------|------|
| 未ログイン | `index.html` へリダイレクト |
| クエリパラメータ `id` がない | `tasks.html` へリダイレクト |
| 指定IDのタスクが存在しない | `tasks.html` へリダイレクト |

### 画面要素

| 要素 | data-testid | 種別 | 説明 |
|------|-------------|------|------|
| タイトル入力欄 | `title-input` | input[text] | 既存値を初期表示 |
| メモ入力欄 | `memo-input` | textarea | 既存値を初期表示（任意） |
| 期限入力欄 | `due-date-input` | input[date] | 既存値を初期表示（任意、YYYY-MM-DD形式） |
| 戻るボタン | `back-button` | button | 保存せずにタスク一覧へ戻る |
| 保存ボタン | `save-button` | button | 変更を保存してタスク一覧へ戻る |

### 保存仕様

保存ボタンクリック時に以下のフィールドを `updateTask()` で更新する。

| フィールド | 処理 |
|---------|------|
| `title` | `titleInput.value.trim()` |
| `memo` | `memoInput.value`（trimなし） |
| `dueDate` | `dueDateInput.value`（YYYY-MM-DD または空文字） |

`completed` フィールドは編集画面では変更しない（一覧画面のチェックボックスのみで操作）。

### 遷移仕様

| 操作 | 遷移先 |
|------|--------|
| 保存ボタンクリック | `tasks.html` |
| 戻るボタンクリック | `tasks.html` |
